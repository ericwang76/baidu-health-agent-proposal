行业痛点

中国互联网医疗市场存在三个结构性矛盾：

矛盾需求侧供给侧现状
信息不对称用户搜索症状，无法判断是否需要就医医生无法触达真正需要服务的患者百度搜索承接大量医疗query，但停留在信息层，未转化为服务
供需错配用户找不到"对的"医生，只能按医院/职称筛选医生在线时间碎片化，等待低质量咨询互联网问诊平台医生空转率高，用户付费意愿低

模式本质：三方价值创造

对用户

  • 搜索即服务——不再需要自己判断"该不该看医生"
  • 零成本体验真人医生（平台补贴）
  • AI预采集信息，医生接手即了解情况，体验远优于传统问诊
  • 从"信息焦虑"到"行动方案"一步到位

对医生

  • 只接高意向、已预筛的患者（不浪费时间在"随便问问"上）
  • 收到结构化病史摘要，30秒判断是否接诊
  • 确定性收入（补贴兜底）+ 后续交易分成
  • 获得百度搜索级别的患者流量曝光

对平台（百度）

  • 流量变现升级：医疗搜索从CPC广告（¥5-50/点击，用户反感）升级为交易抽成（用户满意、可持续）
  • 获客成本极低：搜索流量免费，补贴¥30即获一个付费医疗用户（行业CAC ¥100-300）
  • 数据飞轮：对话数据持续优化模型→转化率提升→单位经济改善
  • 生态锁定：用户形成"百度搜索→百度健康医生"的行为路径，竞对难以复制

核心经济逻辑

单次调度利润 = 交易抽成(GMV × 15-25%) - 医生补贴(¥30) - AI推理成本(≈¥0.1)

规模化利润 = Σ(单次利润) + 用户LTV增量（复诊/慢病管理/药品复购）

关键洞察：

  • AI成本趋近于零：科普问答由AI承担，边际成本≈0，70%的流量免费服务完毕
  • 补贴是获客投资而非运营成本：¥30补贴换取一个LTV ¥450-750的付费医疗用户，ROI 15-25倍
  • 精准调度是利润核心：转化预测模型确保补贴只花在"会成交"的用户身上，而非撒网式补贴
  • 后端变现链条拉长利润：单次问诊→药品→体检→复诊→慢病管理订阅，一次获客多次变现

🤖转化预测模型

决策公式

调度得分 = P(转化) × E(交易额) - C(医生调度成本) - C(补贴) 当 调度得分 > 动态阈值 时 → 触发医生介入

特征体系

特征类别具体信号影响
意图强度具体症状持续时间、已尝试治疗、主动问"怎么治"↑ 正向
紧迫度"今天""马上""越来越严重"↑ 正向
消费能力账号画像、设备型号、搜索消费层级↑ 正向
科室特征皮肤/医美客单高 vs 感冒客单低科室权重
对话行为追问深度、上传图片/报告、回复速度↑ 正向
流失信号长时间未回复、"我再看看"↓ 负向
历史行为过去成交记录、退款率± 双向

模型演进路径

阶段1（0-1万对话）：规则引擎 → 关键词触发 + 对话轮数≥3 + 意图明确 → 调度 阶段2（1万-10万）：LR / GBDT → 基于积累数据训练，规则作为特征之一 阶段3（10万+）：Transformer on Dialogue → 逐轮实时更新预测，深度对话理解

因素含义设计思路

Score(doctor, user) = α×conversion_rate + β×speed_score + γ×price_fit + δ×quality_score - λ×load_penalty - μ×subsidy_cost
